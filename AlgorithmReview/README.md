ตัวอย่างผลลัพธ์ Queue

ผู้ป่วยที่รับบริการแล้ว: P001
ผู้ป่วยที่รับบริการแล้ว: P002
ผู้ป่วยคนถัดไป: P003
จำนวนผู้ป่วยที่ยังรออยู่: 5
สถานะของ Queue หลังจากดำเนินการทั้งหมด: [P003, P004, P005, P006, P007]
Queue ไม่ว่าง
อธิบายว่า Queue ใช้หลักการ FIFO อย่างไร: Queue ใช้หลักการ FIFO (First In First Out) 
หมายความว่าผู้ป่วยที่มาถึงก่อนจะถูกให้บริการก่อน และผู้ป่วยที่มาทีหลังจะต้องรอคิวต่อไป 
เช่น ถ้าผู้ป่วย P001 มาถึงก่อน P002 ดังนั้น P001 จะได้รับบริการก่อน P002 และถ้าผู้ป่วย 
P003 มาถึงหลัง P002 ดังนั้น P003 จะต้องรอคิวต่อไปจนกว่าผู้ป่วย P001 และ P002 
จะได้รับบริการเสร็จสิ้น
